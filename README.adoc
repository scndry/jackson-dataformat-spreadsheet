[quote]
_Let's get rid of the Spread Sh!t._

A https://github.com/FasterXML/jackson[Jackson^] extension module for reading and writing http://officeopenxml.com/anatomyofOOXML-xlsx.php[SpreadsheetML^]footnote:[XLSX -- XSSF (XML SpreadSheet Format)] using https://poi.apache.org/[Apache POI^] and https://www.xml.com/pub/a/2003/09/17/stax.html[StAX^].
Built as a full streaming implementation -- supports streaming, data-binding, and tree-model access just like any other Jackson data format module.

As we are _Object-Oriented Developers_, we map spreadsheets to POJOs -- not the other way around.

NOTE: Also supports the legacy Microsoft Excel formatfootnote:[XLS -- HSSF (Horrible SpreadSheet Format)] through the same API.
However, the XLS path is not based on https://poi.apache.org/components/spreadsheet/how-to.html#event_api[Event API^], so it may not be suitable for very large _.xls_ files.

toc::[]

== Requirements

* Java 8+
* Jackson 2.14.0+ (verified)
* Apache POI 5.1.0+ (verified)

== Quick Start

=== Reading

.input-entries.xlsx
|===
|Column A |Column B

|1 |2 |3 |4
|===

[source,java]
----
@DataGrid
public class Entry {
    private int a;
    private int b;
    // Constructors, Getters, Setters and toString
}
----

[source,java]
----
SpreadsheetMapper mapper = new SpreadsheetMapper();
File file = new File("input-entries.xlsx");

// Single row
Entry value = mapper.readValue(file, Entry.class);
// Entry(a=1, b=2)

// All rows
List<Entry> values = mapper.readValues(file, Entry.class);
// [Entry(a=1, b=2), Entry(a=3, b=4)]

// Specific sheet by name
SheetInput<File> input = SheetInput.source(file, "Entries");
List<Entry> entries = mapper.readValues(input, Entry.class);
----

=== Writing

[source,java]
----
SpreadsheetMapper mapper = new SpreadsheetMapper();
File file = new File("output-entries.xlsx");

// Single object
mapper.writeValue(file, new Entry(1, 2));

// Collection -- element type required (Java type erasure)
List<Entry> values = Arrays.asList(new Entry(1, 2), new Entry(3, 4));
mapper.writeValue(file, values, Entry.class);

// Specific sheet name
SheetOutput<File> output = SheetOutput.target(file, "Entries");
mapper.writeValue(output, values, Entry.class);
----

.output-entries.xlsx
[cols="1,1"]
|===
|a |b |1 |2 |3 |4
|===

Customize header names with `@DataColumn`:

[source,java]
----
@DataGrid
public class Entry {
    @DataColumn("Column A") private int a;
    @DataColumn("Column B") private int b;
}
----

.output-entries.xlsx
[cols="1,1"]
|===
|Column A |Column B |1 |2 |3 |4
|===

== Nested Objects

Spreadsheets are flat, but POJOs are not.

|===
|ID |NAME |ZIPCODE |ADDRESS LINE 1 |ADDRESS LINE 2 |DESIGNATION |SALARY

|1
|John Doe
|12345
|123 Main St.
|Anytown, USA
|CEO
|$300,000.00
|===

[source,java]
----
@DataGrid
class Employee {
    int id;
    String name;
    Address address;
    Employment employment;
    // ...
}

class Address {
    String zipcode;
    AddressLines addressLines;
    // ...
}

class AddressLines {
    String line1;
    String line2;
    // ...
}

class Employment {
    Designation designation;
    long salary;
    // ...
}

enum Designation {
    CEO, CTO, SM, ARCH, INT
}
----

[source,java]
----
// Flat spreadsheet -> nested POJO
Employee value = mapper.readValue(input, Employee.class);

// Nested POJO -> flat spreadsheet
mapper.writeValue(output, values, Employee.class);
----

Deserialized result:

----
Employee(
    id=1
    name=John doe
    address=Address(
        zipcode=12345
        addressLines=AddressLines(
            line1=123 Main St.
            line2=Anytown, USA
        )
    )
    employment=Employment(
        designation=CEO
        salary=300000
    )
)
----

=== Nested List

Nested lists are serialized into multi-row output. _Deserialization is not currently supported._

[source,java]
----
@DataGrid
class Outer {
    @DataColumn("A") int a;
    List<Inner> inners;
    @DataColumn("E") int e;
    // ...
}

class Inner {
    @DataColumn("B") int b;
    @DataColumn("C") int[] c;
    @DataColumn("D") int d;
    // ...
}
----

[source,java]
----
Outer value = new Outer(
        1, Arrays.asList(
            new Inner(2, new int[]{3, 4}, 5),
            new Inner(6, new int[]{7, 8}, 9)),
        10);
mapper.writeValue(output, value);
----

[cols="1a,1a,1a,1a,1a"]
|===
|A |B |C |D |E

|1       |2       |3 |5       |10
|`BLANK` |`BLANK` |4 |`BLANK` |`BLANK`
|`BLANK` |6       |7 |9       |`BLANK`
|`BLANK` |`BLANK` |8 |`BLANK` |`BLANK`
|===

== Annotations

`@DataGrid` and `@DataColumn` control schema generation.

.`@DataColumn` attributes
* `value()`: Column header name.
* `style()`: Cell style name (registered via `StylesBuilder`).
* `headerStyle()`: Style for the header cell only.
* `width()`: Fixed column width.
* `autoSize()`: Auto-adjust width to fit content. _May not be accurate for https://en.wikipedia.org/wiki/Halfwidth_and_fullwidth_forms[fullwidth forms^] like https://en.wikipedia.org/wiki/CJK_characters[CJK characters^]._
* `minWidth()` / `maxWidth()`: Bounds for auto-sized columns.
* `merge()`: Merge cells across rows in nested lists.

=== Attribute Resolution

Column attributes resolve in this order:

. `@DataColumn` on the property
. `@DataGrid` on the declaring class
. `@DataGrid` on the enclosing (parent) class

[source,java]
----
@DataGrid(autoSizeColumn = OptBoolean.FALSE)
class Foo {
    @DataColumn(style = "Foo_a_style")
    int a;
    @DataColumn(headerStyle = "Foo_b_headerStyle")
    int b;
}

@DataGrid(columnHeaderStyle = "Bar_columnHeaderStyle")
class Bar {
    Foo foo;
}

@DataGrid(columnStyle = "Baz_columnStyle", autoSizeColumn = OptBoolean.TRUE)
class Baz {
    Foo foo;
}
----

[source,java]
----
// Inspect the generated schema
SpreadsheetSchema foo = mapper.sheetSchemaFor(Foo.class);
SpreadsheetSchema bar = mapper.sheetSchemaFor(Bar.class);
SpreadsheetSchema baz = mapper.sheetSchemaFor(Baz.class);
----

Resolved attributes:

[cols="1,1a,1a,1a"]
|===
|Column |Style |Header Style| Auto Size

|foo.a
|`"Foo_a_style"`
|`""`
|`FALSE`

|foo.b
|`""`
|`"Foo_b_headerStyle"`
|`FALSE`

|bar.foo.a
|`"Foo_a_style"`
|`"Bar_columnHeaderStyle"`
|`FALSE`

|bar.foo.b
|`""`
|`"Foo_b_headerStyle"`
|`FALSE`

|baz.foo.a
|`"Foo_a_style"`
|`""`

|baz.foo.b
|`"Baz_columnStyle"`

|`FALSE`

== Styling

[source,java]
----
@DataGrid(columnHeaderStyle = "baseHeader")
class Employee {
    Address address;
    Employment employment;
    // ...
}

class Address {
    @DataColumn(value = "ZIPCODE", style = "zipcode")
    String zipcode;
    // ...
}

@DataGrid(columnHeaderStyle = "employmentHeader")
class Employment {
    @DataColumn(value = "SALARY", style = "salary", headerStyle = "salaryHeader")
    long salary;
    // ...
}
----

[source,java]
----
StylesBuilder builder = new StylesBuilder()
        .cellStyle("baseHeader")
            .border().thin()
            .fillForegroundColor(IndexedColors.GREY_25_PERCENT)
            .fillPattern().solidForeground()
            .font().bold().end()
            .end()
        .cellStyle("employmentHeader", /*cloneStyleFrom*/ "baseHeader")
            .fillForegroundColor(198, 239, 206)
            .end()
        .cellStyle("salaryHeader", "baseHeader")
            .fillForegroundColor(0xFFC7CE)
            .font().bold().color(0x9C0006).end()
            .end()
        .cellStyle("salary")
            .dataFormat("[Red][>=100000]$#,##0.00;$#,##0.00")
            .end()
        .cellStyle("zipcode")
            .dataFormat().text()
            .end();

SpreadsheetMapper mapper = SpreadsheetMapper.builder()
        .stylesBuilder(builder)
        .build();
----

.Built-in data format shortcuts
* `general()`: `"General"`
* `text()`: `"@"`
* `numberInt()`: `"0"`
* `numberFloat()`: `"0.00"`
* `numberIntWithComma()`: `"&#35;,##0"`
* `numberFloatWithComma()`: `"&#35;,##0.00"`
* `date()`: `"yyyy-mm-dd"`
* `dateTime()`: `"yyyy-mm-dd hh:mm:ss"`

=== Excel Dates

Excel stores dates as numeric serial values.
`ExcelDateModule` is registered by default, automatically converting between Java date types and Excel serial numbers.

Supported: `java.util.Date`, `java.util.Calendar`, `java.time.LocalDate`, `java.time.LocalDateTime`.

=== Origin

[source,java]
----
// Data grid starts at B2 instead of default A1
SpreadsheetMapper mapper = SpreadsheetMapper.builder()
        .origin("B2")   // or .origin(1, 1) or .origin(new CellAddress("B2"))
        .build();
----

|Column B

=== Header

[source,java]
----
// Default: origin row = header, data starts at origin + 1
SpreadsheetMapper mapper = new SpreadsheetMapper();

// No header: data starts at origin
SpreadsheetMapper mapper = SpreadsheetMapper.builder()
        .useHeader(false)
        .build();

// Toggle on a schema instance
SpreadsheetSchema schema = mapper.sheetSchemaFor(Entry.class)
        .withUseHeader(false);
----

=== Merge Columns

[source,java]
----
@DataGrid
class Order {
    @DataColumn(value = "Order ID", merge = OptBoolean.TRUE)
    int orderId;
    List<Item> items;
    @DataColumn(value = "Total", merge = OptBoolean.TRUE)
    int total;
}

class Item {
    @DataColumn("Product") String product;
    @DataColumn("Qty") int qty;
}
----

[source,java]
----
Order order = new Order(1, Arrays.asList(
        new Item("Apple", 3),
        new Item("Banana", 5)),
        8);
mapper.writeValue(output, order);
----

[cols="1a,1a,1a,1a"]
|===
|Order ID |Product |Qty |Total

|1 (merged)
|Apple
|3
|8 (merged)

|
|Banana
|5
|
|===

Class-level default: `@DataGrid(mergeColumn = OptBoolean.TRUE)`.

=== Column Names

[source,java]
----
// Override globally
SpreadsheetMapper mapper = SpreadsheetMapper.builder()
        .columnNameResolver(prop -> "Overwrite " + prop.getName().toUpperCase())
        .build();
----

Or resolve from a custom annotation:

[source,java]
----
enum ColumnCode {
    A("Code A"), B("Code B");
    String text;
    // ...
}

@Retention(RetentionPolicy.RUNTIME)
@interface NameOf {
    ColumnCode value();
}

@DataGrid
class Entity {
    @NameOf(ColumnCode.A) int a;
    @NameOf(ColumnCode.B) int b;
    // ...
}

ColumnNameResolver byText = ColumnNameResolver.annotated(NameOf.class, ColumnCode::getText);
SpreadsheetMapper mapper = SpreadsheetMapper.builder()
        .columnNameResolver(byText)
        .build();
----

=== Parser Features

[source,java]
----
SpreadsheetMapper mapper = SpreadsheetMapper.builder()
        .enable(SheetParser.Feature.BREAK_ON_BLANK_ROW)
        .build();
----

.`SheetParser.Feature`
* `BLANK_ROW_AS_NULL` (default `true`): Blank rows are deserialized as `null`.
* `BREAK_ON_BLANK_ROW` (default `false`): Stop reading at a blank row.
* `FILE_BACKED_SHARED_STRINGS` (default `false`): Store shared strings on disk via H2 MVStore. Requires `com.h2database:h2` on classpath. See link:PERFORMANCE.adoc[Performance Guide].

== I/O Types

* Reading: `File`, `InputStream`, `SheetInput<T>`, `org.apache.poi.ss.usermodel.Sheet`
* Writing: `File`, `OutputStream`, `SheetOutput<T>`, `org.apache.poi.ss.usermodel.Sheet`

`Sheet` allows fine-grained control via the POI API -- reading multiple sheets from one workbook or writing to an existing template.
